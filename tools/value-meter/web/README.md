# value.macrokit.dev — the value project + Agent Value-Meter live demo

The site has two pages:

- **`/`** — the homepage for *A Mathematical Theory of Value* (the framing, the three
  load-bearing equations, the evidence, honest status, links to the preprint).
- **`/tools/value-meter/`** — the live Value-Meter demo.

A **static, client-only** site. The meter runs the **real** `value_meter.py` in the
browser via [Pyodide](https://pyodide.org) — there is no app server, no upload, and no
API key. A visitor's records are parsed and measured entirely client-side, and the
demo reproduces the paper's published numbers live (e.g. the qwen0.5b/intent example →
`I = 1.4166`, out-of-sample `ΔG = 1.3077`).

## Layout

```
web/
├── build.sh                 # copies the REAL Python source into the meter's py/ and
│                            #   regenerates the example record sets (single source
│                            #   of truth — re-run on every deploy)
├── gen_examples.py          # cached harness runs -> small record-set JSONs
├── public/                  # the deployed static site
│   ├── index.html           # the theory homepage  (served at /)
│   ├── style.css            # shared design system (both pages)
│   └── tools/value-meter/   # the meter demo  (served at /tools/value-meter/)
│       ├── index.html  app.js
│       ├── py/              # value_sim.py + value_meter.py  (copied by build.sh)
│       └── examples/        # real example record sets  (generated by build.sh)
└── deploy/
    ├── deploy.sh            # build -> rsync public/ -> install vhost -> reload nginx
    └── nginx/value.macrokit.dev.conf
```

`tools/value-meter/py/` and `tools/value-meter/examples/` are **generated** by
`build.sh` from the canonical sources (`sim/value_sim.py`,
`tools/value-meter/value_meter.py`, and the cached runs). They are committed so the site
is directly serveable, but `build.sh` overwrites them on every deploy — edit the
canonical sources, not the copies.

## Run locally

```sh
tools/value-meter/web/build.sh                       # refresh py/ + examples/
cd tools/value-meter/web/public && python3 -m http.server 8080
# open http://localhost:8080
```

## Deploy (owner)

```sh
tools/value-meter/web/deploy/deploy.sh
```

One-time prerequisites:
1. **DNS** — an A record `value.macrokit.dev → 3.95.41.84`.
2. **TLS** — after the vhost is first installed: `sudo certbot --nginx -d value.macrokit.dev`.

The deploy script mirrors `studio/deploy/deploy.sh`: it rebuilds the static assets,
rsyncs `public/` to `/macrokit/value-site/public` on the EC2 box, installs the nginx
vhost, and reloads nginx. Override the connection with `MACROKIT_SSH_KEY` /
`MACROKIT_SSH_HOST` / `MACROKIT_REMOTE_ROOT`.

## Why Pyodide (not a JS reimplementation)

Running the actual `value_meter.py` means the demo is provably the same instrument as
the CLI and the self-test — no second implementation to keep in sync, and no risk of
the web numbers drifting from the published ones. The cost is a one-time Pyodide+numpy
download on first load; everything after is instant and offline.
